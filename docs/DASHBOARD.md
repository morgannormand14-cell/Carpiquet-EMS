# Dashboard

Sprint 1 defines the visual and functional requirements. The premium dashboard implementation is scheduled for Sprint 2.

## Required Sections

- Branding and simulation status
- Grid power and target
- Hyper SOC, PV and simulated output
- SolarFlow SOC, PV and simulated output
- Total simulated output
- Balance Index
- Health status
- Historical graphs
- User parameters

## Design Requirements

- mobile-first;
- native Home Assistant cards where practical;
- readable units;
- explicit status text;
- Carpiquet EMS visual identity;
- no dependency on colour alone.
